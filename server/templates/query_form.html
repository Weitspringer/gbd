Query Syntax

Context free language:
S -> (S) | S and S | S or S | C
C -> NOV
N -> L([a-zA-z0-9_]+) (L(R) language resulting of regular expression R)
O -> - (nothing) | < | > | <= | >= | like
V -> L([a-zA-Z0-9_\-\%]+)
If your are searching for Strings, use for example "benchmarks like %graph%"
and if you are searching for numbers use "clauses > 100"
Example of intersection of two conditions:
"clauses > 100 and variables < 50"
